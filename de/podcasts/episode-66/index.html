[00:00] Intro: OpenClaw v2026.6.5-beta.2 monatlicher Cadence-Wechsel, Claude Code .168 Tagelang verzögerter Fix, OpenAI ChatGPT Super-App, Apple WWDC 2026, Anthropic Mythos erweitert, Microsoft MAI in Copilot, Gemma 4 12B auf dem Mac

Das OpenClaw v2026.6.5-beta.2 Prerelease ist das Haupt-Release des Zyklus. Es wechselte die Release-Züge zu einem monatlichen Patch-Nummerierungsschema mit der Juni 2026 Basis bei 5.28. Das Build bündelt den neuen Parallel gebündelten web_search Provider, MCP-Tool-Ergebnis-Erzwingung für Nicht-Text- und Nicht-Bild-Blöcke, Anthropic extended-thinking Wiederherstellung nach Prompt-Cache-Ablauf und einen macOS Node-Mode-Fix. Der Claude-Stack hatte auch einen Ausfall am Freitag, den 5. Juni, für etwa zwei Stunden ab 11:19 Uhr EDT, der die Claude API, Claude Code, claude.ai und Claude Cowork mit erhöhten Fehlerraten traf, die hauptsächlich Opus 4.7 und 4.8 betrafen — Downdetector erreichte nahezu tausend US-Berichte, bevor Anthropic bestätigte, dass das Incident bis Freitagnachmittag behoben war. Claude Code 2.1.168 ist die Reaktion am nächsten Tag: ein gezielter tagelang verzögerter Bug-Fix auf der .167 Basis, der Session-Attachment-, Stream-JSON-Event-Reihenfolge- und Interrupt-Handling-Bugs schließt, von denen mehrere den Fehlermodi entsprechen, die Benutzer während des Ausfallfensters gemeldet haben. Nach dem Harness-Block baut OpenAI ChatGPT vor einem Börsengang im Herbst als Coding-Agents-Super-App neu auf, Apple WWDC 2026 eröffnet mit einem Gemini-gestützten Siri, Anthropic erweitert Project Glasswing auf über 150 Organisationen, Microsoft liefert MAI-Thinking-1 und MAI-Code-1-Flash in GitHub Copilot, und Gemma 4 12B erscheint in Google AI Edge Gallery für Mac als 16GB lokales Multimodal-Modell. Die MCP-Spur ist diese Woche kurz — ein Ein-Absatz-Hinweis, keine Vertiefung. Projektradar deckt A2A v1.0 und den CheetahClaws Python Harness ab.

[02:00] Claude Freitag-Ausfall (5. Juni), Claude Code .168 tagelang verzögerter Bug-Fix, OpenClaw v2026.6.5-beta.2 monatlicher Cadence-Wechsel — Release-Abdeckung

Der Claude-Stack wurde am Freitag, den 5. Juni 2026, ab ca. 11:19 Uhr EDT für etwa zwei Stunden getroffen. Anthropic bestätigte erhöhte Fehlerraten bei Claude API, Claude Code, claude.ai und Claude Cowork. Die Störung betraf hauptsächlich Opus 4.7 und 4.8, und Downdetector erreichte nahezu tausend US-Berichte — vierzig Prozent Claude Chat, dreiunddreißig Prozent Claude Code, zwanzig Prozent die Claude-App. Anthropics Statusseite zeigte, dass das Incident bis Freitagnachmittag behoben war, und die öffentliche Erklärung war, dass die Erfolgsquoten auf erwartete Niveaus zurückgekehrt waren. Der Ausfall ist der realweltliche Grund, warum Claude Code .168 einen Tag später herauskam. Das Release ist ein gezielter Bug-Fix auf der .167 Basis, der Session-Attachment-Probleme, Stream-JSON-Event-Reihenfolge-Regressionen und Interrupt-Handling-Bugs schließt — mehrere davon entsprechen den Fehlermodi, die Benutzer während des Ausfalls gemeldet haben. Die Erkenntnis für die Harness-Schicht ist die operative Reaktionszeit. Ein zweistündiger Ausfall am Freitag, ein Bug-Fix-Release am Samstag und ein Eintrag im Änderungsprotokoll in derselben Woche ist der neue Normalfall für den Agent-Stack.

Claude Code 2.1.168 ist das npm Latest, veröffentlicht am 6. Juni um 23:41 UTC, einen Tag nach der .166 und .167 Release-Welle. Die Version ist aus dem npm Registry und dem Änderungsprotokoll verifiziert. Dies ist ein gezieltes Bug-Fix-Release, kein Feature-Release. Die Release-Notizen beschreiben eine Bereinigungswelle, die Session-Attachment-Probleme, Stream-JSON-Event-Reihenfolge-Regressionen und Interrupt-Handling-Bugs schließt, die gegen die .167 Basis gemeldet wurden. Der Umfang ist wichtig, weil diese drei Oberflächen genau die Stellen sind, wo Hintergrundagentenarbeit stillschweigend schiefgeht. Eine Hintergrundsession, die nicht sauber angehängt wird, ist eine Session, die ihre laufende Aufgabe verliert, und der Fehlermodus ist unsichtbar, bis der Operator sich wieder verbindet und eine leere Aufgabenliste vorfindet. Eine Stream-JSON-Session, die die Event-Reihenfolge falsch behandelt, ist eine Session, die Arbeit mitten im Tool-Aufruf verliert, und der Verbraucher am anderen Ende des JSON-Pipes sieht ein partielles oder dupliziertes Event. Ein Interrupt-Handler, der einen Tastendruck verschluckt, ist eine Session, die aussieht, als ob sie hängt, obwohl sie tatsächlich die Eingabe akzeptiert hat und auf das Modell wartet. Punkt eins sechsundachtzig ist der Bereinigungspass für Punkt eins sechsundsiebzig, und das Team hat den Patch innerhalb eines Tages nach dem Feature-Release herausgebracht. Die Versionsmetadaten sind auch eine Erwähnung wert. Der npm Latest Dist-Tag ist Punkt eins sechsundachtzig, und der npm Stable Dist-Tag bleibt Punkt eins dreiundfünfzig. Diese Lücke zwischen Latest und Stable ist beabsichtigt. Anthropic verwendet den Latest Dist-Tag, um sich durch Bereinigungsreleases vorwärtszurollen, während Stable auf einem bekannmaßenden Build für Fleet-Umgebungen gepinnt bleibt, die es vorziehen, nicht jedem Punkt-Release zu folgen. Das Verhaltensdelta zwischen Punkt eins sechsundsiebzig und Punkt eins sechsundachtzig liegt in der Session-Stabilität, nicht in der Fähigkeit. Hintergrundsessions, die feststeckten, sollten sauber fortgesetzt werden. Stream-JSON-Verbraucher, die abgeschnittene oder duplizierte Tool-Events erhielten, sollten saubere Event-Reihenfolge sehen. Interaktive Benutzer, die am Anfang eines Turns Interrupt drückten und sahen, wie die Session den Tastendruck ignorierte, sollten jetzt sehen, dass der Interrupt akzeptiert wurde.

Die OpenClaw-Linie bewegt sich auf einer anderen Achse. Das am 7. Juni veröffentlichte v2026.6.5-beta.2 Prerelease trägt das neue monatliche Patch-Nummerierungsschema, und die Juni 2026 Basis ist auf fünf Punkt zwei acht gepinnt. Die bedeutsame Änderung ist strukturell. Der Release-Zug ist auf einen monatlichen Cadence umgestiegen, und das Versionierungsschema hat sich geändert. Das nächste stabile OpenClaw-Release ist auf einem monatlichen Cadence, und Betreiber sollten eine neue Form erwarten. Pre-Transition-Tags bleiben kompatibel, was bedeutet, dass nichts beim Upgrade kaputtgeht, und bestehende sechs Punkt eins Installationen funktionieren weiterhin ohne Eingriff. Das Juni Prerelease-Bundle selbst ist dicht. Der Parallel gebündelte web_search Provider ersetzt eine externe Abhängigkeit durch eine In-Process-Implementierung, und der Gewinn ist Latenz und Zuverlässigkeit auf dem Suchpfad. Der gebündelte Provider entfernt einen Netzwerk-Hop und eine Drittanbieter-API-Oberfläche, was bedeutet, dass ein Tool-Aufruf, der Suchergebnisse zurückgibt, nicht mehr davon abhängt, dass ein externer Dienst verfügbar ist. Die MCP-Tool-Ergebnis-Erzwingung behandelt Nicht-Text- und Nicht-Bild-Ergebnisblöcke einheitlich, sodass ein Tool, das eine strukturierte Nutzlast zurückgibt, keinen benutzerdefinierten Adapter mehr benötigt, damit der Agent sie konsumieren kann. Die Anthropic extended-thinking Wiederherstellung nach Prompt-Cache-Ablauf schließt eine Klasse von Wiederherstellungsproblemen, bei denen der Prompt-Cache invalidiert wird und der extended-thinking-Zustand verloren geht. Der macOS Node Mode Fix verhindert eine stille Selbstreconnect weg von einer gesunden direkten Gateway-Session. Stable OpenClaw bleibt bei sechs Punkt eins ab Juni 3.

[12:00] OpenAI ChatGPT „Super-App" — Chat ist tot

Das Financial Times berichtete am 7. Juni, dass OpenAI den größten ChatGPT-Überarbeit seit dem Launch vorbereitet. Das Pitch von innerhalb des Unternehmens ist unverblümt, und ein hochrangiger OpenAI-Mitarbeiter sagte der FT, dass „Chat tot ist." Das neue ChatGPT wird als einheitliche Super-App neu aufgebaut, die Codex, KI-Agents, Bildgenerierung und Drittanbieterdienste einbezieht. Die Produktoberfläche wird auf ein einzelnes Umsatzprodukt verengt, das die Arbeit erledigt, anstatt darüber zu sprechen. Thibault Sottiaux, der OpenAIs Kernprodukt und Plattform leitet, formulierte das Ziel als „persönlichen Agenten, der Ihnen bei allem in Ihrem Leben helfen kann, persönlich oder bei der Arbeit." Der strategische Kontext ist ein IPO-Rennen. Anthropic reichte am 1. Juni vertraulich ein. OpenAI wird erwartet, in den kommenden Wochen zu folgen. Anthropics annualisierter Umsatz erreichte 47 Milliarden Dollar im Mai, gegenüber 30 Milliarden Dollar früher in diesem Jahr, größtenteils durch Claude Code und die Mythos-Vorschau. OpenAI wird von Investoren gesagt, dass es einen klareren Umsatzpfad braucht, und die Super-App ist es. Der Zug erklärt auch die Sora-Herunterfahrung. Im März berichtete das Wall Street Journal, dass OpenAI „Seitenprojekte" wie das eigenständige Sora-Videoprodukt aufgab. Die Super-App-Strategie bestätigt diese Einschätzung. OpenAI konsolidiert seine Oberfläche in ein einzelnes Umsatzprodukt anstatt eines Portfolios von Experimenten. ChatGPT Plugins, der erste Versuch des Unternehmens für diese Konsolidierung im März 2023, blieb nicht hängen. Der Versuch 2026 hat eine andere Grundlage. Codex ist real, agentisches Coding zahlt sich aus, und der Unternehmenskunde hat bereits einen Single-Sign-On-Pfad. Das „Chat ist tot" Framing ist das Marketing-Reset — das vorherige Framing war Assistent, das aktuelle ist Kollege.

[22:00] Apple WWDC 2026 — Gemini-gestütztes Siri

WWDC 2026 eröffnet am 8. Juni um 10:00 Uhr PT mit einer vorab aufgezeichneten Keynote, die aus dem Apple Park gestreamt wird. Dies ist Tim Cooks letzte WWDC als CEO, bevor er die Rolle im September an John Ternus übergibt. Die Schlagzeile ist die Siri-Überarbeitung, die Apple seit WWDC 2024 angedeutet und verzögert hat. Das neue Siri basiert auf einem benutzerdefinierten Gemini-Modell, das gemeinsam mit dem Google Gemini-Team als Teil der Januar 2026 Apple-Google-Partnerschaft entwickelt wurde. Das gemeldete Feature-Set: konversationeller, kontextbewusster, mehrstufiger Aufgabenbearbeitung, App-übergreifende Aktionen und eine eigenständige Siri-App, die direkt mit ChatGPT, Claude und Gemini konkurrieren kann. Bloombergs Mark Gurman berichtet über einen neuen „Visual Intelligence"-Abschnitt in der Kamera-App, der Google Image Search für Objekterkennung verwendet. Berichte deuten auch auf KI-gestützte Fotos-Features, KI-Wandpapiere, die an die Stimmung des Benutzers gebunden sind, erweiterte Genmoji und eine App Store KI-Agent-Integration hin. Die Betriebssystem-Reihe wird als iOS 27, iPadOS 27, macOS 27, watchOS 27 und visionOS 27 ausgeliefert. iOS 27 muss den ersten faltbaren iPhone von Apple, der im September ausgeliefert wird, unterbringen. Das iPhone Fold soll erstmals zwei Apps nebeneinander unterstützen, mit einem iPad-ähnlichen Display im aufgeklappten Zustand.

[32:00] Anthropic Project Glasswing erweitert auf über 150 Organisationen

Anthropic kündigte am 2. Juni an, dass Project Glasswing, sein gemeinsames Branchenprogramm zur Findung und Behebung kritischer Softwareschwachstellen mit KI, auf etwa 150 neue Organisationen in über 15 Ländern erweitert wird. Die Erweiterung deckt Energie, Wasser, Gesundheitswesen, Kommunikation und Hardware ab. Branchen, die in der ursprünglichen 50-Partner-Kohorte, die im April Mythos Preview-Zugang erhielt, nicht „gut vertreten" waren. Neuer Zugang geht an den US-basierten Identitäts- und Sicherheitsanbieter Okta, südkoreanische Unternehmen Samsung, SK Hynix und SK Telecom, die NATO, die Cybersicherheitsagentur der EU ENISA und andere. Die ursprüngliche Kohorte hat Berichten zufolge Mythos verwendet, um über 10.000 hohe oder kritische Sicherheitsmängel zu finden. Anthropic sagt, es arbeite „so schnell wie möglich, um Mythos-Level-Fähigkeiten sicher zu veröffentlichen" an die Öffentlichkeit, aber die öffentliche Veröffentlichung wartet auf „hochrobuste Sicherheitsvorkehrungen", um Missbrauch zu verhindern. Politico berichtete diese Woche, dass Anthropic zugesagt hat, Mythos-Klasse-Modelle allen Kunden „in den kommenden Wochen" verfügbar zu machen. Anthropics IPO-Zeitplan (vertraulich eingereicht am 1. Juni) liegt im gleichen Fenster. Das Unternehmen braucht einen Mythos-Klasse-öffentlichen Modell-Launch und eine Sicherheitsgeschichte, die stark genug für ein S-1 ist, um zu landen. Der Wettbewerbskontext: OpenAI bietet GPT-5.5 Cyber an britische Banken an, zu denen Anthropic bisher den Zugang zu Mythos-Vorschauen blockiert hat. Das britische AI Security Institute testete beide Modelle und berichtete über „ein ähnliches Leistungsniveau." Dieses Ergebnis ist das Signal, dass die Grenze genuin bifurziert. Beide Labore haben ein Cyber-fähiges Modell, beide kontrollieren den Zugang sorgfältig, und beide verwenden Sicherheitsframing zur Differenzierung.

[42:00] Microsoft MAI-Thinking-1, MAI-Code-1-Flash, MAI-Image-2.5

Microsoft nutzte Build 2026 am 2. Juni, um sein erstes hauseigenes fortschrittliches Reasoning-Modell und eine vollständige Pipeline unterstützender Modelle anzukündigen. MAI-Thinking-1 ist ein „mittelgroßes Modell", das Microsoft sagt, stimmt mit führenden Modellen bei wichtigen Software-Engineering-Benchmarks überein. MAI-Code-1-Flash ist als inferenz-effizient positioniert und ist in GitHub Copilot und Visual Studio Code integriert. Das ist der stack-relevanteste Drop für den Agent-Stack, weil MAI-Code jetzt eine Erstanbieter-Microsoft-Option für Code-Mode-Flows im Editor ist, in den die meisten Agents bereits verdrahtet sind. MAI-Image-2.5 (und eine Flash-Variante) behandelt Text-zu-Bild und Bildbearbeitung. MAI-Transcribe-1.5 ist „fünfmal schneller als konkurrierende Modelle" bei Speech-to-Text. MAI-Voice-2 (mit einer Flash-Version „demnächst verfügbar") fügt 15 neue Sprachen und neue Stimmoptionen hinzu. PCMag testete alle vier und nannte die neue MAI-Familie „in Ordnung, und das ist etwa das Beste, was ich über sie sagen kann." Das Reasoning ist konkurrenzfähig, die Bild- und Stimmodelle sind funktional, aber nicht differenziert, und die Erkenntnis ist, dass Microsoft jetzt eine funktionierende hauseigene Modellsuite hat, die OpenAI über Bild-, Stimm- und Code-Pfade ersetzen kann. Diese Substitutionsfähigkeit ist der strategische Punkt. Microsoft ist nicht mehr ausschließlich von OpenAI für die Modellschicht seiner Produktoberfläche abhängig.

[52:00] Gemma 4 12B auf Google AI Edge Gallery für Mac

Google veröffentlichte Gemma 4 12B am 3. Juni 2026 — ein zwölf Milliarden Parameter Open-Weights-Modell mit einer Apache 2.0 Lizenz, das für die lokale Ausführung auf einem Standard-Laptop mit sechzehn Gigabyte VRAM oder Unified Memory entwickelt wurde. Die architektonische Verschiebung ist das Encoder-freie „Unified"-Design. Rohe Audio-Wellenformen und visuelle Patches fließen direkt in den LLM-Backbone ohne sekundäre Verarbeitungsmodule, was bedeutet, dass das Modell hören, sehen und reasoning kann, ohne eine Routing-Schicht. Das Kontextfenster ist 256K Token, mit nativen agentischen Tool-Use-Fähigkeiten und einem schrittweisen Reasoning-Modus. Gemma 4 12B ist sofort auf Hugging Face, Kaggle und durch Google AI Edge Gallery verfügbar, das am selben Tag auf macOS gestartet wurde. Die Begleit-App Google AI Edge Eloquent Diktat ist ebenfalls auf dem Mac verfügbar. Die fünf Google-Modelle, die in AI Edge Gallery für Mac verfügbar sind, sind alle aus der Gemma-Familie, abgestimmt auf Instruct-Verhalten — Instruction-Following anstatt Text-Vervollständigung. Das Encoder-freie Design ist wichtig, weil es den Latenz- und Speicher-Overhead separater Audio- und Vision-Encoder entfernt. Das traditionelle Multimodal-Design leitet Audio und Vision durch dedizierte Encoder, die Embeddings produzieren, und die Embeddings werden dann mit dem Text-Token-Stream verkettet und in den LLM eingespeist. Das Encoder-freie Design überspringt die Encoder-Stufe und speist die rohen Audio- und Vision-Token direkt in den LLM ein, was bedeutet, dass das Modell lernt, Audio und Vision als Teil desselben Token-Streams zu behandeln. Der Latenzgewinn ist real: eine Multimodal-Anfrage zahlt nicht mehr die Encoder-Inferenzkosten, und der Speichergewinn ist real: die Encoder-Gewichte sind aus dem Working Set verschwunden. Das 256K Kontextfenster ist die andere architektonische Wette. Ein zwölf Milliarden Parameter Modell mit einem 256K Kontextfenster ist eine echte Fähigkeit für lokale Agent-Stacks.

[60:00] MCP-Spur (kurzer Hinweis) und Projektradar: CheetahClaws, A2A Protocol v1.0

Ein kurzer MCP-Hinweis diese Woche. OpenAI führt Lockdown Mode und Active Sessions für ChatGPT am 8. Juni ein und bringt zwei Kontosicherheitskontrollen breiter verfügbar — Lockdown Mode begrenzt ausgehende Netzwerkanfragen, um Datenexfiltration von Prompt-Injection-Angriffen zu reduzieren, und Active Sessions ermöglicht es Benutzern zu überprüfen, wo ihr Konto angemeldet ist. Die Kontrollen landen auf persönlichen und Self-Serve-Business-Konten. Dasselbe MCP-Ökosystem, das das Ende-Mai-Audit produzierte, bewegt sich auf der Responseseite schnell, und sowohl die Scanner- als auch die Server-Hardening-Oberflächen erhalten Updates. Das ist ein Ein-Absatz-Hinweis, keine Vertiefung — der Nachrichtenzyklus ist diese Woche modelllastig. CheetahClaws drei Punkt null fünf ist ein Python-nativer Multi-Modell-Agent-Harness von SafeRL-Lab, der als lesbare Alternative zum kompilierten TypeScript-Bundle entwickelt wurde, das die meisten Agent-Harnesses ausliefern. Das Release landete am 4. Juni mit Claude-Code-Style Quiet-Output als Standardverhalten. Die Agent-Loop passt in etwa 740 Zeilen Python, und die Modell-Unterstützungsliste ist breit — Anthropic, OpenAI, Gemini, Kimi, Qwen, Zhipu, DeepSeek, mehrere andere, Ollama, LM Studio und jeden OpenAI-kompatiblen Endpoint. Das Feature-Set deckt Runtime-Tool-Registrierung mit MCP und Git-Plugins, Markdown-Skills für deklarative Fähigkeitsdefinition, einen Aufgaben-Abhängigkeitsgraphen mit Blocks und Blocked-By-Semantik, zwei-Schicht-Kontextkompression, Offline-Voice, Cloud-Session-Sync und Brücken zu Telegram, WeChat, Slack und QQ ab. Das Repo hat über 700 Stars mit Aktivität, die sich auf die Agent-Loop und den Aufgabengraphen konzentriert. Der Kompromiss ist real — Fehlerbehandlung bei Provider-Ausfällen, Retry-Logik bei transienten Tool-Fehlern und Observability-Hooks sind dünner als ein ausgereifter Harness wie Claude Code. Das A2A Protocol erreichte Version eins in 2026 unter der Linux Foundation. Ursprünglich von Google gestartet, wird A2A jetzt zusammen mit MCP regiert. Das Protocol definiert Agent Cards — JSON-Fähigkeits-Manifeste für Agent-Discovery — und eine aufgabenbasierte Statemaschine für langlebige Interaktionen unter Verwendung von JSON-RPC 2.0. Die MCP-versus-A2A-Unterscheidung ist das Schlüssel-Mentalmodell: MCP standardisiert, wie ein Agent sich mit externen Tools, Datenbanken und Datenquellen verbindet; A2A standardisiert, wie Agents miteinander kommunizieren. Das Repository hat über 24.000 Stars und aktive Entwicklung, und das Protocol hat ausreichende Reife erreicht, dass Builder es kennen sollten, wenn sie Multi-Agent-Workflows entwerfen.

[66:00] Praktische Warteschlange

Die praktische Warteschlange diese Woche ist kurz und konkret. Für Claude Code führen Sie den Versionsbefehl aus und bestätigen Sie, dass Sie auf Punkt eins sechsundachtzig sind; wenn Sie eine Hintergrundsession gehalten haben, die Stream-JSON-Stalls oder Interrupt-Probleme auf Punkt eins sechsundsiebzig zeigte, sollte das Upgrade sie beheben. Für Ausfall-Resilienz beobachten Sie die Anthropic-Statusseite und abonnieren Sie den Incident-RSS-Feed, damit Sie lokale Modellfehler mit Plattform-Incidents in Echtzeit korrelieren können, und rotieren Sie alle langlebigen Claude-API-Sessions, die möglicherweise mitten im Aufruf während des Freitag, 5. Juni-Fensters geendet haben. Für OpenClaw entscheiden Sie, ob Sie das Juni Prerelease verfolgen oder Stable auf sechs Punkt eins halten basierend auf Ihrer Version-Pin-Richtlinie. Für OpenAI ChatGPT prüfen Sie eigenständige Produkte, die in die Super-App zusammenfallen werden, und notieren Sie Daten-Export-Pfade für jeden sunset-anfälligen Dienst. Installieren Sie die iOS 27 oder macOS 27 Beta nach WWDC und testen Sie das neue Siri gegen ChatGPT oder Claude für mehrstufige Aufgabenbearbeitung. Für Anthropic überwachen Sie die Mythos-öffentliche Veröffentlichungsankündigung und fragen Sie Ihr Account-Team nach dem Zeitplan. Für Microsoft aktivieren Sie MAI-Code-1-Flash in GitHub Copilot und führen Sie einen Completion-Test gegen Ihr aktuelles Standardmodell durch. Für Gemma 4 12B laden Sie den Checkpoint herunter und führen Sie ihn auf einem 16GB Mac aus, um eine Coding-Aufgabe mit Ihrem aktuellen lokalen Modell zu vergleichen. Für CheetahClaws klonen Sie das Repo und lesen Sie die 740-Zeilen-Agent-Loop. Für A2A lesen Sie die Version-eins-Spezifikation und identifizieren Sie einen Übergabepunkt in Ihrem Workflow, wo Agent Cards eine benutzerdefinierte Integration ersetzen könnten.
