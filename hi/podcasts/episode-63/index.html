Episode 63 — 4 जून, 2026

[00:00] Episode hook

OpenClaw v2026.6.1, Hermes Agent v2026.5.29.2, और Claude Code 2.1.162 एक ही episode window में आ गए हैं। Stable OpenClaw tag v2026.6.1 है, Hermes stable tag v2026.5.29.2 पर बना हुआ है, और latest Claude Code npm latest 2.1.162 है। OpenClaw v2026.6.1 में Workboard orchestration, governed Skill Workshop, SQLite-backed state recovery, और MiniMax M3 provider support शामिल है। Claude Code 2.1.162 में claude agents --json में waiting-for visibility जोड़ी गई है और 2.1.158 से 2.1.162 तक पाँच releases में permission और interrupt fixes का एक batch है। Qwen 3.7 Max और Plus coding-reasoning और multimodal-vision lanes को divide करते हैं। agentmemory आपकी मशीन पर हर agent को एक persistent context layer share कराता है। यह एक 60-मिनट का episode है — existing builder stories रखें और runtime extend करें।

[02:00] OpenClaw v2026.6.1 deep dive — Workboard, Skill Workshop, SQLite recovery, और provider expansion

OpenClaw v2026.6.1 3 जून, 2026 को आया और यह main episode block है। Headline addition है Workboard — orchestration primitives, task-backed board runs, और edit modal में task comments के साथ एक multi-agent planning और run-tracking surface। Workboard सिर्फ एक task list नहीं है; यह एक full coordination surface है जो right agent wiring के साथ task decomposition और worktree-per-task execution drive कर सकता है। Key shift यह है कि OpenClaw single-agent runtime से multi-agent coordination layer की तरफ बढ़ रहा है। Workboard में एक kanban board सिर्फ status tracking नहीं है — यह agent coordination tools के साथ multi-agent planning और run tracking के लिए actual task decomposition drive करता है।

Skill Workshop Control UI में governed skill authoring जोड़ता है। Proposals apply/reject/quarantine controls से flow होते हैं rollback metadata के साथ। Standard skill folders के तहत support files को scanner, hash, और rollback safeguards मिलते हैं। skill_workshop agent tool अब OpenClaw agent toolset का हिस्सा है — एक agent skill proposal initiate कर सकता है, उसे review करवा सकता है, और OpenClaw runtime के अंदर बिना external tooling के approve करवा सकता है। skill_workshop agent tool guarded review flow के through explicit proposals apply, reject, और quarantine कर सकता है। Proposals standard skill folders के तहत approved support files ले जा सकते हैं scanner, hash, और rollback safeguards के साथ। Pending proposals को approval से पहले versioned, dated proposal frontmatter के साथ in-place revise किया जा सकता है। Skill Workshop guidance Codex app-server prompts में भी surface होता है जब skill_workshop available हो, जिसका मतलब है कि Codex run में right moment पर skill governance के लिए prompt कर सकता है। Control UI में अब proposal lists, today actions, revision handoff, searchable file previews, review states, locale coverage, और reusable session routing के साथ एक fuller Skill Workshop flow है।

Agent और CLI runtime recovery तीसरा major improvement area है। Tool calls जो interrupt होते हैं, stale session bindings, compaction handoffs, और media delivery retries सभी more cleanly recover होते हैं। Fixes में interrupted tool calls, stale session bindings, compaction handoffs, और media delivery retries शामिल हैं जो पाँच pull requests में cover हैं। Provider और plugin request paths में और timers, retries, OAuth और device-code lifetimes, media downloads, local service probes, और generated-content polling paths bind की गई हैं before वे एक run hang कर सकें।

iMessage monitor state, inbound queues, और plugin install ledgers SQLite-backed state में चले गए हैं so restarts duplicate filesystem scanning के बिना recover होते हैं। Plugin install index अब SQLite में persisted है so installed package lookup restarts के बाद भी survive होती है कम filesystem scanning के साथ। iOS में hosted push relay defaults, realtime Talk playback, और variable network conditions के तहत more reliable mobile sessions के लिए एक guarded WebSocket ping path जोड़ी गई है। Telegram, WhatsApp, iMessage, Slack, Discord, Microsoft Teams, Google Chat, Google Meet, और iOS realtime Talk के across channel और mobile delivery inbound queues और push relay में fixes के साथ steadier है।

Provider expansion model registry में MiniMax M3 जोड़ता है जिसमें 1,000,000-token context window, sparse attention (MSA) architecture, multimodal capabilities, MiniMax Code, और full API availability है — OpenClaw-managed agents standard provider interface के through बिना custom configuration के route कर सकते हैं। Account OAuth endpoints अब provider metadata में included हैं more complete OAuth flows के लिए। Google और Vertex catalog fixes model browsing और credential-aware catalog loading address करते हैं। OpenRouter अब model metadata caching के लिए SQLite use करता है, जिसका मतलब है कि catalog state restarts survive होती है और catalog metadata के लिए repeated network calls avoid होती हैं। Copilot Claude 1M capabilities recognized हैं। Foundry reasoning alignment Microsoft के Foundry endpoint through reasoning models address करता है। OpenAI response replay guards non-deterministic replay edge cases के against protect करते हैं। Skills, session metadata, gateway runtime state, plugin metadata, memory watchers, और store writes hot paths पर less repeated work करते हैं। Code mode internal namespaces add करता है scoped agent और global sessions के लिए exact namespace tool dispatch के साथ, plus code-mode integrations के लिए MCP API files और docs।

Practical upgrade list: एक Workboard open करें, एक task create करें, और verify करें कि board state gateway restart के across persist करता है; एक Skill Workshop proposal को apply और review through run करें; verify करें कि एक interrupted tool call cleanly recover होता है; एक channel delivery (Telegram या Discord) को mid-delivery interrupt survive करता है check करें; confirm करें कि एक provider path (MiniMax M3 या OpenRouter) correctly load होती है; और new guarded WebSocket ping path के साथ iOS Talk playback test करें।

[26:00] Claude Code 2.1.162 — waitingFor JSON, permission rules, MCP timeout fixes, और interrupt handling

Claude Code 2.1.162, EP059 के 2.1.157 बेसलाइन से पांच रिलीज आगे है। सबसे ऑपरेटर-प्रासंगिक बदलाव यह है कि claude agents --json अब एक waitingFor फील्ड दिखाता है — यह बताता है कि बैकग्राउंड सेशन क्यों रुका हुआ है, चाहे वह permission prompt हो या कोई दूसरा blocker। इससे ऐसा मॉनिटरिंग टूल बनाना संभव हो गया है जो स्टॉल की जांच कर सकता है बिना किसी इंसान को attach करके inspect करने की जरूरत।

WebFetch permission rules अब explicit हैं — deny और ask rules preapproved-host auto-allow से पहले प्राथमिकता रखते हैं, जिससे एक गैप बंद हो गया जहां एक domain preapproved list में हो सकता था और फिर भी explicit rules हो सकते थे जिन्हें उसे block करना चाहिए था। बैकस्लैश और case-variant paths के लिए Windows path handling ठीक किया गया है — Read deny rules अब वास्तव में Glob और Grep results से फाइलों को छुपाते हैं। 1000ms से कम MCP timeout configs अब 1 सेकंड पर floor नहीं होते और हर tool call को abort नहीं करते; वे default या MCP_TOOL_TIMEOUT पर fall through होते हैं, और claude mcp get उन्हें accordingly annotate करता है। Interrupt-at-turn-start fix stream-json और SDK sessions में एक gap बंद करता है जहां एक prompt भेजने के तुरंत बाद Esc दबाने से turn silently drop होता था और बिना किसी feedback के चलता रहता था।

/effort अब पुष्टि करता है जब gewählte level नए sessions के लिए default के रूप में बनी रहती है। Remote Control एक persistent footer pill के रूप में session link के साथ दिखता है — यह अब एक startup message की तरह scroll away नहीं होता। Windsurf-to-Devin rebrand /ide, /terminal-setup, और /scroll-speed में प्रतिबिंबित है। जब config directory read-only या unwritable हो तब silent startup hang अब एक blank screen के बजाय diagnostic दिखाता है। शांत startup groups notices को severity और session info के अनुसार व्यवस्थित करता है और announcements एक launch प्रति एक ही line share करते हैं।

LSP tool का workspaceSymbol operation अब एक query parameter स्वीकार करता है और इसे language server को pass करता है। classifier side-queries और MCP server descriptions में emoji truncation boundaries के पास 400 no low surrogate in string API errors ठीक किए गए हैं। कई claude agents UI issues हल की गई हैं: status detail अब wide terminals पर 60–120 कॉलम पर truncate नहीं होती, लंबे session names अब 40 कॉलम पर truncate नहीं होते, attach अब background-service restart के बाद पहले try पर session list पर वापस नहीं उछलता, Ctrl+V image paste dispatch input और session reply box में काम करता है, और ← के साथ backgrounding अब silently conversation नहीं खोता जब background service शुरू नहीं हो पाती। Failed turns अब एक multi-line red error block के बजाय एक compact warning line दिखाते हैं।

व्यावहारिक test list: एक background session पर claude agents --json run करें और waitingFor check करें; WebFetch explicit deny behavior verify करें; एक MCP server को 1000ms से कम timeout के साथ test करें और confirm करें कि यह claude mcp get में annotate है; एक stream-json session में एक turn के बिल्कुल शुरुआत में एक interrupt send करें और Interrupted feedback verify करें।

[40:00] Qwen 3.7 Max और Plus — coding-reasoning vs multimodal-vision split on the agent frontier

Qwen 3.7 19 मई, 2026 को दो अलग-अलग variants के रूप में आया। Max reasoning, coding, और math को target करता है — Codex और Claude Code के primary models के समान lane। Plus multimodal और vision tasks को target करता है। दोनों open-weight हैं और OpenRouter API और local deployment tools के माध्यम से उपलब्ध हैं। Qwen का agentic positioning मतलब यह है कि models tool use और agent-computer interaction के लिए डिज़ाइन की गई हैं, सिर्फ single-turn completion के लिए नहीं।

OpenClaw, Codex, और Claude Code operators के लिए, Qwen 3.7 Max coding-heavy workloads के लिए एक routing option है। agents के लिए जिन्हें images, documents, और visual inputs handle करने की जरूरत है, Qwen 3.7 Plus pure coding models से एक अलग lane भरता है। यह split model routing decisions के लिए मायने रखती है — आप इस आधार पर चुनते हैं कि task मुख्य रूप से code generation है या multimodal understanding।

Try it: Qwen 3.7 quantization को Ollama या LM Studio के माध्यम से pull करें और अपने current coding agent model के विरुद्ध code generation quality की तुलना करें। multimodal tasks के लिए, एक document या image input के साथ OpenRouter API के माध्यम से Qwen 3.7 Plus try करें।

[51:00] agentmemory — फुल एजेंट स्टैक के लिए शेयर्ड पर्सिस्टेंट कॉन्टेक्स्ट

agentmemory हर उस एजेंट की ब्लैंक-स्लेट समस्या को हल करता है जिसे आप चलाते हैं। जब Claude Code, Codex, Hermes, और OpenClaw सभी पोर्ट 3111 पर एक ही मेमोरी सर्वर शेयर करते हैं, तो पिछले हफ्ते के काम का कॉन्टेक्स्ट इस हफ्ते के सेशन में जीवित रहता है। सिस्टम iii इंजन का उपयोग करता है, यह ग्लोबल npm पैकेज के रूप में इंस्टॉल होता है, और एजेंट के आधार पर नेटिव प्लगइन्स, हुक, या MCP के जरिए कनेक्ट होता है।

सबसे ठोस यूज़ केस एक डेवलपर है जो कई एजेंट्स में एक ही कोडबेस पर काम करता है — हाई-लेवल रिफैक्टरिंग के लिए Claude Code, शेल और फाइल ऑपरेशन्स के लिए Codex, मल्टी-स्टेप टास्क्स को ऑर्केस्ट्रेट करने के लिए Hermes। शेयर्ड मेमोरी लेयर के बिना, हर एजेंट कोल्ड शुरू होता है। agentmemory के साथ, Claude Code के आखिरी सेशन का कॉन्टेक्स्ट बताता है कि Codex आगे क्या करता है।

इंस्टॉल करें, डेमो चलाएं, अपने एजेंट से कनेक्ट करें, और दो सेशन्स में रिकॉल को वेरिफाई करें।

[61:00] CodeGraphContext v0.4.13 — एजेंट्स और डेवलपर्स के लिए कोड ग्राफ़

CodeGraphContext कोड को एक ग्राफ डेटाबेस में इंडेक्स करता है और CLI और MCP सर्वर दोनों के माध्यम से ग्राफ को एक्सपोज़ करता है। डुअल-मोड डिज़ाइन का मतलब है कि एक इंडेक्स ह्यूमन एक्सप्लोरेशन और एजेंट टूल कॉल्स दोनों की सर्विस करता है। इंडेक्सिंग इंजन tree-sitter नोड्स का उपयोग करके कोड पार्स करता है और कंपोनेंट्स का ग्राफ बनाता है — फंक्शन्स, क्लासेस, मेथड्स, पैरामीटर्स, इनहेरिटेंस रिलेशनशिप्स, फंक्शन कॉल्स, और इम्पोर्ट्स। CLI मोड डेवलपर्स को ग्राफ के खिलाफ नैचुरल लैंग्वेज क्वेरीज़ रन करने देता है। MCP मोड get_callers, get_callees, get_class_hierarchy, और get_call_chain जैसे टूल्स को MCP प्रोटोकल के माध्यम से एक्सपोज़ करता है। लाइव फाइल वॉचिंग फाइल्स बदलने पर रियल टाइम में ग्राफ को अपडेट करता है। प्री-इंडेक्स्ड बंडल्स आपको बिना इंडेक्सिंग के प्रसिद्ध रिपॉज़िटरीज़ को तुरंत लोड करने देते हैं। ग्राफ उन बड़े कोडबेस पर टूल कॉल ओवरहेड को कम करता है जहाँ टेक्स्ट सर्च को दर्जनों फाइल्स में चलाने की जरूरत होती है। इसे अपने एजेंट से कनेक्ट करें और MCP इंटरफेस के जरिए वही सवाल पूछें ताकि एजेंट के जवाब की तुलना CLI से मिले जवाब से की जा सके।
