[02:30] एजेंट-स्टैक रिलीज़ रीडआउट — OpenClaw v2026.5.18 OpenClaw v2026.5.18 एक व्यापक होस्ट रिलीज़ है, लेकिन पहला बिल्डर-फेसिंग बदलाव प्लगइन आकार है। रिलीज़ defineToolPlugin के साथ-साथ openclaw plugins init, openclaw plugins build, और openclaw plugins validate जोड़ती है टाइप किए गए सरल टूल प्लगइन्स के लिए जेनरेटेड मैनिफेस्ट मेटाडेटा, वैकल्पिक डिक्लेरेशन्स, और कॉन्टेक्स्ट फैक्ट्रीज़ के साथ। यह छोटे प्लगइन काम को हैंड-रिटन मैनिफेस्ट ग्लू पर कम निर्भर बनाता है। यह deprecation के लिए भी मायने रखता है: रिलीज़ पुराने rich-message producer APIs जैसे legacy interactive और Slack directive paths को deprecated के रूप में मार्क करती है जबकि चैनल presentation capability limits जोड़ती है, इसलिए प्लगइन authors को renderer वास्तव में क्या दिखा सकता है इसके लिए एक स्पष्ट कॉन्ट्रैक्ट मिलता है।

ब्राउज़र में बदलाव छोटे लेकिन ऑटोमेशन विश्वसनीयता के लिए महत्वपूर्ण हैं। स्नैपशॉट अब पेंडिंग और हाल ही में हैंडल किए गए मॉडल डायलॉग्स को सतह पर लाते हैं, एक्शन्स blockedByDialog लौटा सकती हैं जब कोई मॉडल खुलता है, और browser dialog --dialog-id पेंडिंग डायलॉग का जवाब दे सकती है। यह ब्राउज़र एजेंट्स में एक सामान्य failure mode को बदलता है: इसके बजाय कि क्लिक चुपचाप फेल हो क्योंकि अलर्ट या confirm डायलॉग ने पेज को seized कर लिया, ऑटोमेशन लेयर डायलॉग को स्टेट के रूप में रिप्रेजेंट कर सकती है और एजेंट को एक स्पष्ट अगला एक्शन दे सकती है।

Gateway स्टार्टअप और प्रॉक्सी व्यवहार भी बदलते हैं। स्टार्टअप लॉगिंग और plugin-service स्टार्टअप अब चैनल sidecars के साथ ओवरलैप करते हैं जबकि /readyz sidecar gating को प्रिज़र्व करते हैं, और restart traces probe, config, runtime, और resource-count costs को एट्रिब्यूट करते हैं बिना readiness semantics को बदले। ऑपरेटर्स के लिए, उपयोगी हिस्सा सिर्फ तेज़ ready time नहीं है; यह बेहतर प्रमाण है जब restart धीमा हो। रिलीज़ HTTPS मैनेज्ड forward-proxy endpoints और scoped proxy.tls.caFile trust भी जोड़ती है, जो deployments को TLS-inspected या private proxy paths के माध्यम से रूट करने का एक स्वच्छ तरीका देती है बिना प्रॉक्सी कॉन्फ़िगरेशन को global trust decision में बदले।

QA-Lab का काम असामान्य रूप से महत्वपूर्ण है। OpenClaw पहले घंटे के 20-टर्न और वैकल्पिक 100-टर्न runtime parity सिनारियोज़, openclaw qa suite --runtime-parity-tier, openclaw qa coverage --tools के माध्यम से टूल फिक्सचर कवरेज, लाइव runtime token-efficiency आर्टिफैक्ट्स, और standard Codex-vs-Pi tier में आवश्यक OpenClaw dynamic runtime-tool drift के लिए hard gate जोड़ता है। सरल शब्दों में: प्रोजेक्ट रिलीज़ चेक्स जोड़ रहा है जो runtime behavior, टूल vocabulary, टोकन यूज़, और टूल कवरेज की तुलना करते हैं इसके बजाय smoke test को पर्याप्त मानने के। एजेंट होस्ट्स के लिए यह सही दिशा है, जहां regressions अक्सर "एजेंट ने गलत टूल का उपयोग किया" के रूप में दिखाई देते हैं "प्रोसेस क्रैश हुई" के बजाय।

Android Talk Mode को भी एक बड़ा runtime बदलाव मिलता है। Android ऐप Talk Mode को realtime Gateway relay वॉइस सेशन्स में स्विच करता है streaming microphone input, realtime audio playback, टूल-रिज़ल्ट bridging, और on-screen transcripts के साथ। यह मोबाइल वॉइस को एक सरल input/output wrapper से एक सक्रिय सेशन में बदलता है जो टूल रिज़ल्ट्स को Gateway के माध्यम से वापस ले जा सकता है। टेस्ट करने का जोखिम interruption और latency है: streaming वॉइस सेशन्स को clean cancellation, transcript alignment, और टूल-रिज़ल्ट टाइमिंग की जरूरत है जो यूज़र को stale output सुनने न दे।

फिक्सेस वह जगह है जहां कई production अपग्रेड्स रिलीज़ को महसूस करेंगे। जेनरेटेड मीडिया completions अब Telegram forum topics में वापस आती हैं requester-agent handoff में topic IDs को प्रिज़र्व करके। इमेज मेटाडेटा प्रोबिंग unrecognized bytes पर बाहरी decoder delegates को invoke करने से बचती है, और Sharp native imaging tools, ImageMagick, GraphicsMagick, या ffmpeg के fallbacks के साथ इंस्टॉल किया गया है। Discord वॉइस सेशन्स OpenAI realtime और prebuffer assistant playback के साथ follow-up टर्न्स सुनना जारी रखती हैं choppy starts को कम करने के लिए। Message/TTS directives message-tool sends delivery paths तक पहुंचने से पहले लागू होते हैं, इसलिए opt-in rooms को raw tags के बजाय वॉइस नोट्स मिलते हैं।

OpenClaw के Codex ऐप-सर्वर रिपेयर्स मिक्स्ड एजेंट स्टैक्स के लिए विशेष रूप से प्रासंगिक हैं। Current inbound image attachments queued runs से पहले hydrate होती हैं ताकि Responses-बैक्ड एजेंट्स चैनल इमेज्स को native vision input के रूप में प्राप्त करें। Native code mode code-mode-only को फोर्स किए बिना उपलब्ध रहता है, जो OpenClaw dynamic टूल टर्न्स को ऐप-सर्वर bridge के माध्यम से complete करने देता है। Network access sandboxed Codex code-mode टर्न्स के लिए प्रिज़र्व रहता है जब OpenClaw sandbox outbound egress की अनुमति देता है। Per-agent code-mode config schema, runtime catalog activation, और model payload filtering में सम्मानित किया जाता है। Restricted chat या sender policy अब restricted टर्न्स के लिए native code, app, environment, और user MCP surfaces को डिसेबल करके fail closed होती है। पैटर्न स्पष्ट है: Codex integration policy, media, sandboxing, और model/runtime selection के आसपास कसा जा रहा है।

OpenClaw v2026.5.18 के लिए माइग्रेशन नोट्स ठोस हैं। न्यूनतम समर्थित Node.js 22 लाइन 22.19 तक बढ़ जाती है, Pi पैकेजेस 0.75.1 में जाते हैं, और Docker/Podman builds को OPENCLAW_IMAGE_APT_PACKAGES को प्राथमिकता देनी चाहिए जबकि OPENCLAW_DOCKER_APT_PACKAGES legacy fallback के रूप में रहता है। Obsidian skill अब third-party obsidian-cli के बजाय आधिकारिक obsidian CLI को टारगेट करती है। Repo-local Codex closeout review skill और helper का नाम बदलकर autoreview किया गया है। Channel/plugin authors को पुराने rich-message controls के सभी जगह समान रूप से render होने की अवधारणा को अपनाने से पहले deprecated message producer surfaces और presentation capability limits की जांच करनी चाहिए।

[17:30] Agent-stack रिलीज रीडआउट — Codex rust-v0.131.0 Codex rust-v0.131.0 मैचिंग CLI-साइड रिलीज है। TUI अब डेटा-ड्रिवन सर्विस-टियर कमांड, ब्लेंडेड टोकन यूसेज, परमिशन और अप्रूवल मोड, इफेक्टिव वर्कस्पेस रूट्स, और रेस्पॉन्सिव मार्कडाउन टेबल्स एक्सपोज करता है। यह इंटरफेस का काम लगता है, लेकिन यह day-to-day ऑपरेशंस बदल देता है: एक लंबी रन के दौरान, ऑपरेटर देख सकता है कि एजेंट वास्तव में किस परमिशन और अप्रूवल एन्वेलप का उपयोग कर रहा है और कौन से वर्कस्पेस रूट्स एक्टिव हैं, कॉन्फिग फाइल्स और मेमोरी से फैक्ट्स रीकंस्ट्रक्ट करने के बजाय।

Mentions व्यापक होते जा रहे हैं। @ सर्च अब एक पिकर में फाइल्स, डायरेक्टरीज, प्लगइन्स, और स्किल्स को कवर करता है, जो app-server प्लगइन मेटाडेटा द्वारा बैक किया जाता है। यह इंटरैक्शन मॉडल को बिल्डर्स के वास्तविक सोचने के तरीके के करीब लाता है: कॉन्टेक्स्ट में लाने की चीज एक फाइल, डायरेक्टरी, स्किल, या प्लगइन हो सकती है, और एजेंट सरफेस को ये अलग-अलग डिस्कवरी लेन नहीं बनानी चाहिए। रिस्क कॉन्टेक्स्ट ब्लोट है, इसलिए व्यावहारिक सिफारिश है कि पिकर का उपयोग बल्क इम्पोर्ट टूल की तरह इलाज करने के बजाय सबसे छोटा उपयोगी आर्टिफैक्ट अटैच करने के लिए करें।

प्लगइन वर्कफ्लो भी आगे बढ़ते हैं। Codex marketplace CLI कमांड, वर्जन-अवेयर शेयरिंग, शेयर चेकआउट, स्पष्ट शेयर्ड-वर्कस्पेस बकेट्स, और डिफ़ॉल्ट-एनेबल्ड प्लगइन हुक्स जोड़ता है। यह प्लगइन्स को ढीले लोकल फोल्डर्स के बजाय मैनेज्ड डेवलपमेंट आर्टिफैक्ट्स की तरह व्यवहार करने की दिशा में एक कदम है। माइग्रेशन वॉच आइटम हुक ट्रस्ट और स्कोप है: डिफ़ॉल्ट-एनेबल्ड हुक्स पावरफुल हैं, इसलिए प्लगइन प्रोवेनेंस, वर्कस्पेस शेयरिंग, और वर्जन गेट्स ज्यादा मायने रखते हैं क्योंकि वर्कफ्लो स्मूद होता जाता है।

रिमोट वर्क 0.131.0 का एक बड़ा हिस्सा है। रिलीज daemon-मैनेज्ड codex remote-control, रनटाइम एनेबल और डिसेबल APIs, स्टेटस रीड्स, रजिस्ट्री-बैक्ड और कॉन्फ़िगर्ड रिमोट एनवायरनमेंट्स, और रिमोट एनवायरनमेंट्स और desktop-owned कॉन्फिग नेमस्पेस के लिए app-server/API कॉन्ट्रैक्ट्स जोड़ती है। यूजर्स के लिए, क्षमता रिमोट एजेंट वर्क है जिसे इमीडिएट टर्मिनल के बाहर शुरू, मॉनिटर, या मैनेज किया जा सकता है। Codex को इंटीग्रेट करने वाले बिल्डर्स के लिए, API कॉन्ट्रैक्ट्स मायने रखते हैं क्योंकि रिमोट एनवायरनमेंट्स को ड्यूरेबल आइडेंटिटी, लाइफसाइकल, स्टेटस, और कॉन्फ़िगरेशन बाउंड्रीज की जरूरत होती है बजाय best-effort शेल प्रोसेस के।

Python SDK अब openai-codex / openai_codex है, जिसमें पिन्ड रनटाइम-जेनरेटेड टाइप्स, कंकरेंट टर्न रूटिंग, अप्रूवल मोड्स, और इंटीग्रेशन कवरेज हैं। यह Python एप्लिकेशन्स को Codex टर्न्स को चलाने और CLI को टेक्स्ट-ओनली सबप्रोसेस की तरह इलाज करने के बजाय ID द्वारा नोटिफिकेशन प्राप्त करने का एक क्लीनर तरीका देता है। महत्वपूर्ण मैकेनिज्म टर्न रूटिंग है: कंकरेंट एजेंट वर्क को IDs और स्ट्रक्चर्ड इवेंट्स की जरूरत होती है ताकि अप्रूवल्स, टूल एक्टिविटी, और आउटपुट वायर्स क्रॉस न करें।

Codex codex doctor भी जोड़ता है जो रनटाइम, auth, टर्मिनल, नेटवर्क, कॉन्फिग, और लोकल स्टेट में डायग्नोस्टिक्स के लिए है। यह वह कमांड है जो फेल्ड अपग्रेड्स के दौरान अपने आप को चुकाता है। यह पूछने के बजाय कि फेलियर एक टर्मिनल इश्यू है, एक पुराना auth टोकन है, एक नेटवर्क पॉलिसी है, या एक लोकल स्टेट डेटाबेस प्रॉब्लम है, CLI एक जगह पर support-ready एविडेंस इकट्ठा कर सकता है। रिलीज app-server और लोकल स्टेट स्टार्टअप को भी safer बनाती है SQLite डेटा प्रिजर्व करके, जब स्टेट नहीं खुल सकता तब फेल क्लोज करके, रिकवरी पाथ्स जोड़कर, और ऑप्शनल मेटाडेटा सिंक फेलियर्स को soften करके।

बग-फिक्स ब्लॉक वास्तविक फेलियर मोड्स को हार्डन करता है। Windows sandbox बिहेवियर deny-read रूल्स, scoped write रूट्स, ineffective फायरवॉल पॉलिसी, और PowerShell एज केसिस के आसपास सुधार करता है। मैनेज्ड रीड रिस्ट्रिक्शन्स परमिशन एस्केलेशन से सर्वाइव करते हैं, और workspace-root परमिशन प्रोफाइल रेज़ोल्यूशन साफ हो जाता है। Git और auth रिलायबिलिटी root worktree हुक्स, helper कमांड्स में repo हुक और fsmonitor कॉन्फिग को इग्नोर करके, लोकल MCP OAuth कॉलबैक्स को बाइंड करके, और superseded लॉगिन टोकन्स को रिवोक करके सुधरती हैं। रिमोट और Windows क्लीनअप को लॉन्गर exec-server ट्रांसपोर्ट टाइमआउट्स, quieter taskkill, और non-queued प्लगइन रीड्स मिलते हैं। रिलीज एक फ्लैशी कैपेबिलिटी के बारे में कम और long CLI सेशन्स और रिमोट सेशन्स को observable, recoverable, और safer बनाने के बारे में ज्यादा है।

[29:30] GitHub Copilot एजेंट्स — रिमोट स्टीयरिंग, सस्ते मॉडल्स, और Actions रिपेयर GitHub की 18 मई की Copilot अपडेट्स hosted कॉडिंग एजेंट्स के कहां जा रहे हैं इसका एक उपयोगी स्नैपशॉट है। Copilot CLI सेशन्स के लिए रिमोट कंट्रोल अब GitHub Mobile, github.com, VS Code, और JetBrains पर सामान्य रूप से उपलब्ध है। मैकेनिक्स स्पष्ट हैं: copilot --remote के साथ शुरू करें, सेशन के अंदर /remote on से एनेबल करें, या Copilot सेटिंग्स फाइल में remoteSessions सेट करें। एक बार अटैच होने पर, रिमोट सरफेस सेशन एक्टिविटी स्ट्रीम कर सकता है, क्यूd इनपुट स्वीकार कर सकता है, परमिशन प्रॉम्प्ट्स का जवाब दे सकता है, एक सेशन रोक सकता है, और यूजर को ऑरिजिनल टर्मिनल से दूर काम स्टीयर करने दे सकता है।

यह CLI-एजेंट वर्क की शेप बदलता है। एक लोकल टर्मिनल अभी भी execution एंकर है, लेकिन सुपरविजन सरफेस मूव कर सकता है। मशीन को ऑनलाइन रहना चाहिए, और GitHub के डॉक्स लॉन्गर रन के लिए /keep-alive को कॉल आउट करते हैं। सेशन्स यूजर-स्पेसिफिक हैं, और Business या Enterprise यूजर्स को एडमिनिस्ट्रेटर्स की जरूरत हो सकती है रिमोट कंट्रोल और CLI पॉलिसीज को एनेबल करने के लिए। वे कॉन्स्ट्रेंट्स महत्वपूर्ण हैं: रिमोट स्टीयरिंग मैजिक बैकग्राउंड कंप्यूट नहीं है; यह एक रिमोट कंट्रोल प्लेन और पॉलिसी गेट्स वाला एक लाइव लोकल सेशन है।

GitHub ने Claude Haiku 4.5 और GPT-5.4-mini के साथ Copilot क्लाउड एजेंट मॉडल चॉइसेस को भी विस्तृत किया है, प्रत्येक 0.33x मल्टीप्लायर पर लिस्टेड है। यह कॉडिंग एजेंट्स के लिए सही प्रोडक्ट दिशा है क्योंकि हर delegated जॉब को सबसे स्ट्रॉन्ग मॉडल की जरूरत नहीं होती। सीधे dependency bumps, छोटे test फिक्सेस, lint फेलियर्स, या मैकेनिकल एडिट्स एक सस्ते मॉडल पर चल सकते हैं, जबकि डिज़ाइन-हैवी या ambiguous बदलाव एक स्ट्रॉन्गर का उपयोग कर सकते हैं। डॉक्स कहते हैं कि मॉडल सेलेक्शन सपोर्टेड एंट्रीपॉइंट्स से उपलब्ध है जैसे किसी इश्यू को असाइन करना, पुल रिक्वेस्ट कमेंट में @copilot का ज़िक्र करना, एजेंट सरफेस से शुरू करना, GitHub Mobile, या Raycast; जहां कोई पिकर नहीं है, Auto का उपयोग किया जाता है।

कोपायलट का तीसरा बदलाव फेल हो रहे GitHub Actions के लिए वन-क्लिक रिपेयर है। वर्कफ़्लो रन लॉग्स पेज से, Copilot Business और Enterprise सब्सक्राइबर Fix with Copilot पर क्लिक कर सकते हैं; क्लाउड एजेंट विफलता की जांच करता है, ब्रांच में एक फिक्स पुश करता है, और उपयोगकर्ता को समीक्षा के लिए टैग करता है। तकनीकी रूप से, यह एक मजबूत संदर्भ बंडल वाला एक एजेंट एंट्रीपॉइंट है: फेल हो रहे जॉब लॉग्स, ब्रांच स्टेट, रिपॉजिटरी इंस्ट्रक्शंस, और क्लाउड डेवलपमेंट एनवायरनमेंट। व्यावहारिक बिल्डर का निहितार्थ यह है कि CI विफलताएं चैट में मैन्युअल रूप से लॉग पेस्ट किए बिना एक डेलीगेटेबल वर्क आइटम बन जाती हैं। जोखिम है समीक्षा अनुशासन: एक पुश किए गए ब्रांच फिक्स को अभी भी कोड समीक्षा, परीक्षण सत्यापन, और इस बात पर ध्यान देने की आवश्यकता है कि क्या एजेंट ने सही डिज़ाइन के बजाय ग्रीन CI के लिए ऑप्टिमाइज़ किया है।
